# magick-feature-flags v1.4.0 Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Address every P0/P1/P2 finding from the 2026-04-16 audit (concurrency, security, correctness, coverage) and ship a hardened v1.4.0 of the gem.

**Architecture:** Ten sequential phases, each independently commitable. Phase 0 backfills missing specs so subsequent refactors have a safety net. Later phases introduce locks, fork hooks, lossless import/export, Admin UI hardening, Pub/Sub validation, and docs polish. Each phase ends with a passing suite + green rubocop + commit.

**Tech Stack:** Ruby 3.3, RSpec 3.12, Rubocop 1.50, Redis (optional), ActiveRecord (optional), Rails ≥ 6 (optional).

---

## Conventions

- **Commit style:** `[type] Description`. Types: `[fix]`, `[feature]`, `[update]`, `[test]`, `[docs]`, `[ci]`.
- **After every task** run: `bundle exec rspec` (must pass) + `bundle exec rubocop lib/... spec/...` on the touched files (must not introduce a new offense class).
- **Never skip the RED step** for new code. If an existing spec already covers a refactor, note that in the step.
- **Commit cadence:** one commit per numbered task unless explicitly bundled.

### Task 1.2: Mutex around `Magick.features` hash — DEFERRED

**Status:** Deferred. Under MRI Ruby 3.3 the GVL makes `Hash#[]=` atomic per operation. Attempting to reproduce a race with 8 writers × 2000 inserts against 8 concurrent iterators produced zero failures. Forcing snapshot semantics would require returning `@features.dup` from every `Magick.features` call, adding an allocation on every hot-path `Magick.features[name]` lookup (typically 1-2 per `enabled?`). Not worth the cost without evidence of a real race.

A regression guard (`spec/magick/features_concurrency_spec.rb`) is in place. If this is ever run on JRuby / TruffleRuby and fails, the fix is a single `Mutex` around register/reset.

```ruby
def self.import(data, registry)
  Array(data).each_with_object({}) do |payload, imported|
    name = payload[:name] || payload['name']
    feature = Magick::Feature.new(name, registry,
                                  display_name: payload[:display_name],
                                  group: payload[:group])

    feature.status = payload[:status] if payload[:status]
    feature.set_value(payload[:value]) if payload.key?(:value)

    (payload[:targeting] || {}).each do |key, value|
      apply_targeting(feature, key.to_sym, value)
    end

    (payload[:variants] || []).each do |v|
      feature.add_variant(v[:name], weight: v[:weight], value: v[:value])
    end

    (payload[:dependencies] || []).each do |dep|
      Magick::FeatureDependency.add_dependency(name, dep)
    end

    imported[name] = feature
  end
end

def self.apply_targeting(feature, key, value)
  case key
  when :user then Array(value).each { |id| feature.enable_for_user(id) }
  when :excluded_users then Array(value).each { |id| feature.exclude_user(id) }
  when :tag, :tags then Array(value).each { |t| feature.enable_for_tag(t) }
  when :excluded_tags then Array(value).each { |t| feature.exclude_tag(t) }
  when :group then Array(value).each { |g| feature.enable_for_group(g) }
  when :excluded_groups then Array(value).each { |g| feature.exclude_group(g) }
  when :role then Array(value).each { |r| feature.enable_for_role(r) }
  when :excluded_roles then Array(value).each { |r| feature.exclude_role(r) }
  when :percentage_users then feature.enable_for_percentage(value)
  when :percentage_requests then feature.enable_for_percentage_of_requests(value)
  when :ip_address then feature.enable_for_ip_addresses(*Array(value))
  when :excluded_ip_addresses then feature.exclude_ip_addresses(*Array(value))
  when :date_range then feature.set_date_range(value[:start], value[:end]) if value.is_a?(Hash)
  when :custom_attributes then Array(value).each { |rule| feature.add_custom_attribute_rule(rule) }
  when :complex_conditions then feature.set_complex_conditions(value)
  end
end
```

```ruby
def pick_variant_bucket(digest_hex, total_weight)
  # 64-bit hash → drop the top bits that would introduce modulo bias.
  modulus = 2**64
  limit = modulus - (modulus % total_weight)
  loop do
    value = digest_hex[0, 16].to_i(16)
    return value % total_weight if value < limit

    digest_hex = Digest::SHA256.hexdigest(digest_hex) # re-hash on the rare case we land in the bias window
```
